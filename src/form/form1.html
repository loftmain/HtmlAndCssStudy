2.4.1 Form 기본 속성

폼 동작 방식

  1. 웹 페이지에 있는 form에 데이터를 입력
  2. 웹 페이지 내 액션이 일어나게 되면 데이터는 웹 서버로 이동
  3. 웹 서버는 데이터를 처리하기 위해 APP을 호출, 이때 APP은 물리적으로 별도의 서버일 수 있음
  4. 필요에 따라 APP은 DB로 데이터를 전송, 이때 DB는 물리적으로 별도의 서버일 수도 있음
  5. DB에서 CRUD 작업이 일어나고 작업 결과를 APP으로, WEB으로 전송
  6. 웹 서버는 받은 결과를 Client 브라우저에게 보냄
  7. 사용자 브라우저는 Response 받은 페이지를 렌더링하여 사용자에게 보여줌

Form의 속성은 다음과 같이 이루어져 있습니다.

action
입력 값을 전송할 페이지를 나타냅니다.
method
폼의 데이터를 전송할 방법을 정의합니다. method의 속성에는 get과 post가 있습니다. get은 웹 서버에 데이터를 요청할 때 사용하며, 주소에 데이터를 입력하는 방식이고, post는 파일을 올리거나, 보안이 필요한 데이터를 전송할 때 등 사용됩니다. 또한, 주소에 입력 내용이 나타나지 않도록 합니다.
name
폼 이름을 지정합니다.

2.4.2 input의 기본 속성

Input은 사용자가 다양하게 폼 태그에 입력할 수 있는 공간을 만들어 주고, 사용자에게 정보를 입력받습니다.

Input의 속성은 아래 표와 같이 이루어져 있습니다.

type
태그 모양을 다양하게 변경할 수 있습니다. Type에서는 text, radio, checkbox, password, button 등을 지정할 수 있습니다.
name
태그 이름을 지정합니다.
readonly
태그를 읽기전용으로 합니다.
maxlength
최대 글자 수를 지정합니다.
required
필수 태그로 지정됩니다. 필수 태그를 입력하지 않고, submit 버튼을 누르면 에러가 떠 데이터가 전송되지 않습니다.
autofocus
웹 페이지가 로딩 되면 이 속성을 지정한 태그로 포커스가 바뀝니다.
placeholder
입력할 값에 대한 힌트를 줍니다.
pattern
정규표현식을 사용하여 특정 범위 내의 유효한 값을 입력 받을 때 사용합니다.

2.4.3 버튼, 이미지 버튼

일반적인 버튼을 만드려면 type 속성 값을 button으로 지정합니다. 폼을 서버에 보내는 데 사용하는 전송 버튼을 만들고 싶으면 type 속성 값을 submit으로 지정합니다.
전송 버튼에 이미지를 넣고 싶은 경우에는 type 속성 값을 image로 지정하면 됩니다.

2.4.5 날짜 및 시간

날짜를 입력할 폼을 만들 때는 type 속성 값을 date로 지정해줍니다. min과 max로 날짜의 범위도 정해 줄 수 있습니다.
시간을 입력할 폼을 만들 때에는 type속성 값을 time으로 지정해줍니다.

그 밖에도 날짜와 시간과 관련된 폼들이 있는데 아래 표에서 살펴보겠습니다.

  • 연도, 월 표시: month
  • 연도, 주 표시: week
  • 날짜와 시간 함께 표시: datetime
  • 날짜와 시간 함꼐 표시(로컬): datetime-local

2.4.6 범위 및 수치, 컬러

범위 안에서 값을 설정할 수 있는 슬라이드 바 형식의 폼을 만들고 싶을 때에는 type의 속성 값을 range로 정해줍니다.
수를 정할 때에는 type의 속성 값을 number로 정해줍니다. 숫자형을 기본으로 합니다.
min은 최소값, max는 최대값, value는 기본값, step은 이동할, 변경될 크기를 의미합니다.

색을 선택할 수 있는 폼을 만들고 싶다면 type 속성 값을 color로 지정해줍니다.

2.4.8 tel, url, email

이메일 주소를 입력 할 폼을 만들 때는 type의 속성 값을 email로 지정합니다. 웹 페이지 주소를 입력 할 폼을 만들 때는 type의 속성 값을 url로 지정합니다. 전화번호를 입력 할 폼을 만들 때는 type의 속성 값을 tel로 지정합니다.
